# SNHU-Portfolio

## Functionality

This project reads a list of items, each separated by a newline character. It then stores each item and its frequency as key-value pairs in a .dat file. Menu options are prompted to the user, which they can use to view the data in a variety of ways.

## Where I Excelled

I believe I did particularly well in documenting the code. Each function is paired with a header comment describing it and in-line comments are used to further explain functionality.

## Potential Enhancements

One way that this code could be improved is by storing the key-value pairs as a binary object inside of the .dat file. This would help to ensure security. Further action can be taken such as encrypting the binary data before storing it.

## Difficult Sections

The most difficult part of writing this program was probably dealing with the files. I overcame this issue by consulting external sources such as 'https://w3schools.com' or YouTube for assistance in troubleshooting errors and best practices when handling filestreams.

## Transferable Skills

Some of the most transferable skills I obtained in this project include the practice of object-oriented programming and documentation. In previous assignments, my instructor brought up that though I was using an object-oriented language, my code was more akin to a procedural application. After receiving this feedback, I did some research regarding this issue, and to my surprise realized that I had been doing a lot of work incorrectly. Documentation is also an important part of programming, especially when dealing with a foreign language like I was during this course. Having the ability to see the basic functionality of your function at a glance (as achieved b y my header comments) helps to streamline the entire development process.
